## 二、通用面积计算公式

本公式适用于任何商务酒店/综合体的任务书，不局限于本项目。

### 2.1 输入参数（从任务书读取）

```
S_total    = 地上总面积（㎡）      ← 任务书
N_rooms    = 酒店客房总数（间）    ← 任务书
N_hotelF   = 酒店客房层数          ← 任务书
N_officeF  = 办公层数              ← 任务书
N_podiumF  = 裙房层数（通常4F）    ← 任务书
N_basement = 地下层数（通常1F）    ← 任务书
room_w     = 客房开间（m）         ← 通常4.2
room_d     = 客房进深（m）         ← 通常9.5
corridor_w = 走廊净宽（m）         ← ≥1.5
```

### 2.2 核心筒面积公式

```python
core_stairs = 6.5 * 2.6 * 2          # 消防楼梯2座 ≈ 34㎡
core_fire_elev = 2.6 * 2.6 * 2       # 消防电梯2台 ≈ 14㎡
core_pass_elev = 2.4 * 2.4 * 配比      # 客梯台数
core_hall = 15                         # 电梯厅+走道
core_pipe = 8                          # 管井
core_front = 12                        # 防烟前室
core_total = core_stairs + core_fire_elev + core_pass_elev + core_hall + core_pipe + core_front
# ≈ 100~110㎡
```

### 2.3 酒店标准层推导公式

```python
# 每间客房摊分面积(含走廊+核心筒)
room_per_unit = 5                      # 每个单元5间（两排+中廊）
unit_width = room_w * room_per_unit   # 5*4.2 = 21m
unit_depth = room_d * 2 + corridor_w  # 9.5*2+1.5 = 20.5m
unit_area = unit_width * unit_depth   # 21*20.5 ≈ 430㎡

# 每层客房数
rooms_per_floor = N_rooms / N_hotelF  # 207/10 ≈ 20间/层

# 需要的单元数
units_needed = ceil(rooms_per_floor / (room_per_unit * 2))  # 2个单元（双侧各10间）

# 酒店标准层面积
hotel_floor_w = unit_width * units_needed + core_width  # 总宽
hotel_floor_d = unit_depth                               # 总深（固定）
hotel_floor_area = hotel_floor_w * hotel_floor_d          # ≈ 1,050㎡
```

### 2.4 办公标准层推导公式

```python
# 剩余面积优先分配
remaining = S_total
remaining -= hotel_floor_area * N_hotelF    # 扣除酒店
remaining -= average_podium_per_floor * N_podiumF  # 扣除裙房

# 办公每层面积
office_per_floor = remaining / N_officeF

# 办公标准层边长（正方形最优）
office_side = sqrt(office_per_floor)  # ≈ 38m
```

### 2.5 裙房基底面积推导公式

```python
# 受密度和退线双重约束
site_area         # 用地面积
density_max = 0.35
max_building_footprint = site_area * density_max  # 基底不能超此值

# 退线后可用范围
usable_area = (site_width - setbacks) * (site_depth - setbacks)

# 裙房基底 = min(密度限制, 退线范围, 功能需求)
podium_area = min(max_building_footprint, usable_area)
podium_area = min(podium_area, (remaining / N_podiumF) * N_podiumF)
```

### 2.6 通用配平算法

```python
def balance_area(S_total, site_w, site_d, N_rooms, N_hotelF, N_officeF, N_podiumF):
    """
    输入: 地上总面积, 场地宽深, 客房数, 各层数
    输出: 各标准层面积
    """
    # 1. 酒店面积（按客房推导）
    hotel_floor = calc_hotel_floor(N_rooms, N_hotelF)
    hotel_total = hotel_floor * N_hotelF
    
    # 2. 裙房面积（按密度约束）
    max_density = site_w * site_d * 0.35
    podium_floor = min(max_density, sqrt(S_total * 0.3))
    podium_total = podium_floor * N_podiumF
    
    # 3. 办公面积（剩余）
    office_total = S_total - hotel_total - podium_total
    office_floor = office_total / N_officeF
    
    # 4. 验证
    total = hotel_total + podium_total + office_total
    error = abs(total - S_total) / S_total
    
    return {
        "hotel_floor": hotel_floor,
        "hotel_size": f"{round(sqrt(hotel_floor))}m×{round(sqrt(hotel_floor))}m",
        "office_floor": office_floor,
        "office_size": f"{round(sqrt(office_floor))}m×{round(sqrt(office_floor))}m",
        "podium_floor": podium_floor,
        "podium_size": f"{round(sqrt(podium_floor))}m×{round(sqrt(podium_floor))}m",
        "total": total,
        "error": f"{round(error*100,1)}%"
    }
```

## 三、设计流程

```
[开始]
  │
  ▼
① 读取任务书（PDF/Word）
  │ 提取关键参数: S_total, N_rooms, N_hotelF, N_officeF, N_podiumF
  │ 提取约束条件: 限高, 退线, 密度, 容积率
  │
  ▼
② 通用面积配平（调用公用公式）
  │ 输出: 三种标准层面积 + 校验
  │ 验算: 密度≤35%、容积率≤4.0、高度≤99.3m
  │
  ▼
③ 确定标准层尺寸
  │ 酒店: 从客房数推导
  │ 办公: 从剩余面积推导
  │ 裙房: 从密度约束推导
  │ 核心筒: 约10×10.4m（1-26F贯通）
  │
  ▼
④ 确定层高参数
  │ 裙房层高: 4.2m（任务书/规范）
  │ 办公层高: 3.6m（规范最小值）
  │ 酒店层高: 3.8m（规范最小值+吊顶）
  │ 高度校验: 总和 ≤ 限高
  │
  ▼
⑤ 设计三个差异化方案
  │ A 围合内院式: 裙房L形，塔楼分两侧，中央围合庭院
  │ B 线性退台式: 裙房长条展开，酒店退台
  │ C 集中裙房式: 裙房大底盘居中，双塔对称
  │
  ▼
⑥ 生成前校验坐标
  │ 红线范围: 相对偏移原点

  │ 酒店: 全在红线内？
  │
  ▼
⑦ 调用 rhino MCP 生成
  │ 1. 创建母图层 A/B/C
  │ 2. 画控制线（红线+退线）
  │ 3. 创建裙房（1-4F）+ 功能子图层

  │ 7. 标注方案名称

  ▼
⑧ 生成后校验
  │ 面积、红线层、高度、密度、容积率、颜色
  │
  ▼
⑨ 输出完整设计包
  │ 模型: Rhino 3dm（三个方案+控制线）
  │ 数据: 面积配平表
  │ 规范: 校验报告
  │
  ▼
```

## 九、常见错误对照表

| 错误 | 原因 | 修正方法 |
|:-----|:-----|:---------|
| 面积不对 | 生成前没算面积 | 先算后建，允许±5% |
| 超红线 | 没校验边界 | 建前先算XY范围 |
| 塔楼悬空 | 底部16.8m但裙房没覆盖到XY | 裙房XY必须包容办公XY |
| 塔楼齐平边缘 | 和裙房边缘对齐 | 四周至少内缩3m |
| BC和A一样 | 用了相同的相对坐标 | 每次重算三个偏移 |
| 颜色不对 | 没对照色码表 | 建中查表 |
| 坐标爆炸 | `ox+275000`写错 | 用相对ox的坐标 |
| 核心筒在外面 | 核心筒没在塔楼内部 | 居中放置 |
| Y读为0 | Rhino BoundingBox bug | 关注Z值连续性 |
