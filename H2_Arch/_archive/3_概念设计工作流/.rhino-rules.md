# 建筑类型学推理与通用几何生成协议 V2.0
# Typological Reasoning & Generic Massing Protocol

---

## 【核心原则】
AI 在读取任何任务书后，必须依次经过三层推理引擎，严禁跳过类型学诊断直接启动几何生成。

---

## Layer 1：类型学诊断树（The Diagnoser）

### 1.1 诊断输出规范
AI 读完任务书后，必须首先输出以下结构化的类型学诊断报告，然后才能进入设计阶段：

```
=== 类型学诊断报告 ===
建筑类型：          [酒店/公寓/度假村/办公/综合体/...]
定位级别：          [经济型/中档/中高档/奢华]
总建筑面积：        [XX,XXX ㎡]
场地特征：          [城市核心区/郊区/山地/滨海/...]
核心设计矛盾：      [至少列出 2 条，如：多业态垂直声学冲突 / 
                     日照朝向与面宽最大化矛盾 / 
                     景观面最大化与红线约束冲突]
功能组团划分：      [将总层数归并为 3-4 个巨型组团，如：
                     裙房组团 1-4F / 办公组团 5-15F / 客房组团 16-26F]
推荐适用母题：      [列 2-3 个，如：Monolith / Terrace / Interlock]
========================
```

### 1.2 建筑类型与行业公理（诊断依据）

| 类型 | 行业公理 | 核心矛盾锚点 |
|:----|:--------|:-----------|
| **商务酒店** | 下公上私，下闹上静 | 垂直声学阻尼（KTV/宴会 vs 客房）；四流线分流；核心筒垂直交通效率 |
| **高端公寓** | 面宽最大化，户户朝南 | 日照阴影线控制；开间进深比（≥1:1.5）；标准层重复率 |
| **野奢度假村** | 景观最大化，分散式布局 | 地形等高线单向展开；视线扇面无遮挡；低密度零扰 |
| **总部办公** | 高效标准层，灵活分隔 | 核心筒偏置或外挂；标准层进深 ≤18m；立面统一模数 |
| **商业综合体** | 多业态水平串联 | 人流动线组织；中庭空间序列；后勤与客流绝对隔离 |

### 1.3 功能组团划分规则

AI 必须将任务书中的楼层按照功能类型归并为巨型组团，而不是逐层处理。组团划分原则：

1. **同一功能的连续楼层必须归为一个组团**（如 6F-15F 办公 → 办公组团）
2. **功能转换楼层必须作为独立的"转换节点"**，允许在此处发生退台/悬挑/结构转换
3. **每个组团内部的外墙线默认 100% 垂直平齐**
4. **组团数量 ≥2 组，≤4 组**（不含地下和屋顶设备层）

## Layer 2：母题语法库（The Motif Library）

### 2.1 三大核心母题

AI 必须从以下母题库中选择 2-3 个作为方案的几何生成策略。每个方案必须严格绑定其中一种母题，禁止在一个方案内混用多种母题。

#### 母题 A：Monolith（极简巨构）

| 字段 | 内容 |
|:----|:-----|
| **锚定参考** | 密斯·凡·德·罗 - 西格拉姆大厦；SOM - 约翰·汉考克中心；安藤忠雄 - 光的空间 |
| **核心特征** | 外立面 100% 垂直平齐；结构柱网外露或幕墙统一；整体性是最高准则 |
| **适用条件** | 城市核心区；限高严格；甲方要求"稳重高档"；高层建筑（≥20F） |
| **体块逻辑** | 获取最大配平面积生成刚性外壳 → 内部通过 BooleanDifference 挖中庭/入口/天井减面积。外壳外皮在任何楼层都不允许出现凹凸进退。 |
| **面积消纳策略** | 差值全部转化为内部通高中庭、入口凹洞、天井。严禁在外立面产生锯齿。 |
| **塔楼策略** | 塔楼与裙房外皮完全对齐。顶部允许做 ≤3m 的阴角刻槽作为收头。不鼓励悬挑。 |

#### 母题 B：Terrace（阶梯错叠）

| 字段 | 内容 |
|:----|:-----|
| **锚定参考** | BIG - 8 House / Via 57 West；藤本壮介 - 东京公寓；MVRDV - 书山 |
| **核心特征** | 多个巨型组团之间发生整体性跌落；退台面转化为露台/绿化/公共空间；单向退缩，禁止散乱 |
| **适用条件** | 沿主干道；有景观面要求；需要公共空间补偿；中高层建筑 |
| **体块逻辑** | 锁定主立面边界（通常为东/南），所有面积差沿次要方向单向退缩。退让幅度 ≥3m/次，退台面自动标记为 Terrace_Zone。 |
| **面积消纳策略** | 每一级退台在下方组团顶部暴露的水平面转化为露台。退台方向必须一致，禁止东西交错退让。 |
| **塔楼策略** | 塔楼保持与裙房同等退缩方向，形成自下而上的连续退台语言。允许在退台面做 2-3m 悬挑观景平台。 |

#### 母题 C：Interlock（异质穿插）

| 字段 | 内容 |
|:----|:-----|
| **锚定参考** | OMA - 中央电视台总部；扎哈·哈迪德 - 北京大兴机场；蓝天组 - 宝马世界 |
| **核心特征** | 裙房与塔楼作为独立几何体互相咬合；悬挑作为结构叙事；强烈的视觉张力与地标性 |
| **适用条件** | 地标性项目；转角地块；甲方要求"先锋感"；不限高或不严格 |
| **体块逻辑** | 裙房和塔楼各自建立独立的几何中心轴。两者在 Z 轴上允许物理交叉咬合。面积的微调通过调整咬合深度逼近。 |
| **面积消纳策略** | 塔楼体块从裙房体内"穿出"，咬合区的重叠面积计入裙房或塔楼。悬挑产生的面积增量由对侧配重/收缩抵消。 |
| **塔楼策略** | 顶部楼层允许大尺度悬挑（≤4.5m）作为天际线标志性收头。悬挑方向单一，避免四面悬挑造成结构扭曲。 |

### 2.2 母题选择规则

```
IF 建筑类型 = 酒店 AND 定位 ≥ 中高档 AND 限高严格：
    推荐母题 = [Monolith, Terrace, Interlock]
    
IF 建筑类型 = 公寓 AND 面宽要求高：
    推荐母题 = [Terrace, Monolith]
    排除 Interlock（面宽损失大）

IF 建筑类型 = 度假村 AND 低密度分散：
    推荐母题 = [Terrace]
    排除 Monolith（过于集中）和 Interlock（过于激进）

IF 建筑类型 = 总部办公 AND 标准化要求高：
    推荐母题 = [Monolith, Interlock]
    
IF 建筑类型 = 综合体 AND 地标性要求高：
    推荐母题 = [Interlock, Terrace]
```

### 2.3 组团化体量生成规范

无论选择哪种母题，体块生成必须遵守以下规范：

1. **严禁逐层堆叠**。每个组团生成一个完整体块，组团内部所有楼层共用同一外轮廓。
2. **组团数量 2-4 个**。超出 4 个组团需要说明理由。
3. **每个体块必须指派功能标签**（如：裙房组团、办公组团、客房组团）。
4. **核心筒作为独立的贯通体块**，在所有组团中保持完整贯通。

## Layer 3：刚性合规器（The Complier）

### 3.1 面积配平校验

```
计算逻辑：
  组团总面积 = Σ(每层目标面积)   ← 从任务书读取
  体块总面积 = Σ(每个组团体块 BoundingBox 算出的面积)
  误差率 = |体块总面积 - 组团总面积| / 组团总面积
  要求：误差率 ≤ 5%
```

### 3.2 规范检查（按类型加载）

| 类型 | 适用规范 | 检查项 |
|:----|:--------|:-------|
| 酒店 | JGJ62-2014 / GB 51165 | 客房净面积 ≥25㎡；走廊净宽 ≥1.5m；层高 ≥3.3m；无障碍客房 2 间 |
| 公寓 | GB 50096 | 套型面积；日照时数；采光系数；面宽比 |
| 办公 | JGJ 67 | 标准层进深 ≤18m；净高 ≥2.6m；防火分区面积 |
| 通用 | 各地规划条例 | 密度 ≤35%；容积率 ≤4.0；限高；退线；绿地率 |

### 3.3 红线校验

- 所有体块的 XY 坐标必须在退线范围内
- 超限时自动修正：优先缩减裙房边界，再缩减塔楼边界
- 如果三次修正后仍超限 → 切换母题

### 3.4 碎边消灭法案

- 凡相邻两层面积差 ≤150㎡ 或进退幅度 ≤2m 的，禁止在外立面产生锯齿
- 差值一律通过内部核心筒偏心、走廊宽度调节、中庭扩缩消化
- 退台仅允许在组团分界处发生（≥3m）

### 3.5 超限回退协议

```
IF 误差率 > 5% OR 规范校验不通过:
    Step 1: 调整组团内部核心筒位置（偏心 ≤2m）
    Step 2: 调整组团之间的退台/悬挑幅度
    Step 3: 如果前两步仍不满足 → 切换母题重做
```

## Post-Step：后合理化说明（Post-Rationalization）

AI 完成体块生成后，必须输出以下后合理化说明，解释每个体块动作对应的任务书条款：

```
=== 后合理化说明 ===
方案 [A/B/C] —— [母题名称]

[体块动作 1] → [任务书条款]
  [具体描述]

例：
1F 外壳南侧 100㎡ 入口凹洞 → 1F 大堂面积 2,550㎡ 相比 3F 2,650㎡ 减少 100㎡
  → 转化为两层通高的主入口落客门廊，明确主入口位置

3F 西侧 800㎡ 退台 → 4F 宴会厅 1,850㎡ 相比 3F 2,650㎡ 减少 800㎡
  → 转化为 3F 餐饮层的室外观景露台，增加高溢价商业面积

16F 四周内缩 → 客房标准层从办公段 1,350㎡ 退台至 1,210㎡
  → 退台面形成 16F 麻将客房的专属空中花园
========================
```

## V3.0 扩展规则

### 规则三：塔楼弹性铁律与悬挑约束

1. **序列刚性**：1F-4F 裙房、5F-15F 办公、16F 麻将房、17F-25F 客房、26F 云顶的垂直分布顺序禁止变动。
2. **悬挑弹性机制**：允许在满足 0% 面积误差前提下执行悬挑。单向悬挑深度原则上不超过 **3.0m**，云顶等特殊标志性悬挑最大可达 **6.0m** 以上。执行悬挑时，必须自动触发相应面积补偿。

### 规则九：拓扑与声学防御物理法则

1. **强噪声源下沉**：2F KTV 刚性锁定，利用与 16F+ 客房之间长达 13 层的空间深度实现震动衰减。
2. **阻尼缓冲隔离**：5F 办公平面的声学构造与刚度进行针对性加固，作为防御 4F 宴会厅动态阻尼层。

### 规则十：几何外包络控制协议

废弃逐层计算和零碎退台逻辑。方案生成必须锁定以下两种核心母题之一：

#### 方案 B：Terrace — 大尺度单向阶梯（城市绿毯）

| 条款 | 内容 |
|:----|:------|
| **角点锁定** | 东南角点刚性锁定，沿东、南城市界面 1F-26F 保持绝对垂直平齐，严禁锯齿状碎边 |
| **高宽比拉伸** | 塔楼长宽比（`tower_aspect_ratio`）强制设定为 **≥1.6:1**，确保形体扁平挺拔 |
| **单向西侧大退台** | 禁止逐层退台，采用**组团叠落**。4F 顶部向西侧（内院方向）执行极深退台（**≥15m**），将所有面积收分转化为巨型空中生态花园 |

#### 方案 C：Interlock — 极简双折体与硬核悬挑（异质穿插）

| 条款 | 内容 |
|:----|:------|
| **晶体合并** | 5F-25F（含16F）在几何生成时**完全合并**为一个一气呵成的垂直完整巨型玻璃晶体，关闭该区段内所有的收分与退台函数 |
| **非对称咬合** | 晶体塔楼与 1F-4F 实体基座进行**错位咬合**，底层可局部脱开形成大尺度架空 |
| **云顶硬核反冲** | 26F 云顶组团独立计算，顺应塔楼长边方向向主干道侧强制执行单向硬核大悬挑（**≥6.0m**），形成强烈视觉张力 |

#### 方案 B/C 通用约束

- 所有体块 XY 坐标必须在退线范围内
- 面积配平误差 ≤ 5%（V3.0 目标：0%）
- 多个方案在 X 方向并排排列，相邻间距 ≥120m
- 图层命名遵循 Rhino 图层规范（方案名称为前缀）

## 通用附录（不依赖项目类型）

### A. 单位铁律
所有坐标值使用毫米（mm）。1m = 1000mm。

### B. 图层命名规范
```

├── 控制线
```
