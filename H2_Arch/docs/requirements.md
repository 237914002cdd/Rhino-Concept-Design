# H2_Arch 概念设计工作流 — 产品需求

## 项目概述

- **一句话描述**：基于 RhinoMCP 的建筑概念设计 AI 工作流，从任务书到三维体块方案的设计助手
- **目标用户**：方案建筑师（偏 AI 设计工作流方向）
- **核心解决的问题**：将概念设计阶段的推理→面积配平→体块生成→规范校验流程自动化，减少重复手动操作，让建筑师聚焦设计决策

## 用户场景

### 场景 1：建筑师拿到新项目任务书
- **用户**：方案建筑师
- **行为**：将任务书关键参数填入 `project_config.json` → 运行面积配平脚本 → 检查误差率
- **期望结果**：在 1 分钟内得到合乎规范的各标准层面积

### 场景 2：生成三个比选方案
- **用户**：方案建筑师
- **行为**：确认面积配平后 → 启动 Rhino MCP → 用 RhinoScript/Python 生成三个体块方案
- **期望结果**：三个差异化布局的 3D 体块模型，带正确图层和颜色

### 场景 3：方案自我校验
- **用户**：方案建筑师（或 AI 本身）
- **行为**：体块生成后 → 运行校验脚本 → 自动检查面积/红线/规范
- **期望结果**：清晰的 PASS/FAIL 报告，超限时给出修正建议

## 业务规则

1. 面积配平误差必须 ≤ ±5%
2. 所有体块必须在红线范围内
3. 塔楼必须在裙房范围内（四周内缩 ≥3m）
4. 核心筒必须贯通所有楼层且位于塔楼内部
5. 一个方案只能使用一种母题，禁止混用
6. 组团数量必须是 2-4 个（不含地下和屋顶设备层）
7. 同一组团内部各层外轮廓必须垂直平齐

## 非功能需求

- **性能**：面积配平计算应在 1 秒内完成，Rhino 体块生成应在 30 秒内完成
- **兼容性**：支持 Rhino 8 + RhinoMCP 插件 + uvx rhinomcp 协议
- **可扩展性**：新建筑类型只需新增配置模板，无需修改核心算法
