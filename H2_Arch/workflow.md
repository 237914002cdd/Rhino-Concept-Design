## H2_Arch 概念设计工作流 V4.0 — 完整执行指引

### 完整流程概要

```
① 环境准备（防重叠）→ ② 读输入 → ③ 类型学诊断
→ ④ 面积配平+校验（PASS才能继续）
→ ⑤ 数学排布（所有项目通用）
   ├─ 功能块定位：同类用行列公式，异类≥5m间距
   └─ 高度赋值：按功能定层高/层数
→ ⑥ 分类图层 + 色码（所有项目通用）
   ├─ 按功能细分图层（如公建1F_温泉SPA / 独栋客房）
   └─ 色码对照功能（深蓝/紫/蓝/橙/棕/浅棕/米色）
→ ⑦ Rhino生成 + 校验（所有项目通用）
   ├─ 面积回检：每块实测vs目标，误差≤±5%（不通过退回⑤）
   ├─ 重叠检测：XY投影0重叠
   ├─ 高度校验：最高点≤限高
   └─ 目录结构：_rhino_gen.py + src/ + output/ + _archive/
```

### 铁律

1. **类型学优先**：Step ③ 未输出诊断报告，禁止进入 Step ④
2. **先算后建**：Step ④ FAIL 禁止进入 Step ⑤
3. **防重叠**：Step ① 必须检查模型位置，偏移后再生成
4. **轮次递增**：每次新生成 R 编号 +1，旧模型保留
5. **母题纯净**：一个方案只使用一种母题，禁止混用
6. **两次失败停机**：连续两次生成失败 → 输出错误分析，停止
7. **每步有输出**：排布→图层→生成，每步完成后验证再进下一步

### 目录结构规范

每个项目实例必须按以下结构组织文件：

```
projects/<项目名>/
├── _rhino_gen.py           # Rhino 脚本（根目录，方便运行）
├── config.json             # 项目配置
├── src/                    # 源代码（生成器/报告等）
│   ├── gen_everything.py
│   ├── gen_rhino_json.py
│   └── gen_report.py
├── output/                 # 生成物（报告/json/模型引用）
│   ├── Report.xlsx
│   └── _rhino_objects.json
└── _archive/               # 旧版本/不需要的文件
    └── *.3dm / *.py
```

**规则：**
- 核心执行脚本（`_rhino_gen.py`）放根目录方便定位
- 所有生成器/工具放 `src/`
- 输出文件（报告、JSON 数据）放 `output/`
- 旧模型、临时文件、废弃脚本放 `_archive/`
- 临时生成的 `Report_TEMP.xlsx`、`build_rhino.py` 等用完即删，不进目录

**Why：** 实践发现项目根目录混入临时文件、旧模型、输出文件后无法维护。清理后结构清晰且有复用价值。
**How to apply：** 每个项目实例建好目录结构后再开始工作，避免中间混乱。

### 关键原则

- **不预设建筑形态**：Step ③ 诊断决定策略
- **数据驱动排布**：Step ⑤ 是所有项目通用的核心步骤——功能块清单 → 行列公式定位 → 高度赋值 → 分图层 → 生成 → 校验
- **两步验证**：Step ④ 算前校验 + Step ⑦ 生成后校验
- **退线从红线图读取**：不是固定数值

### 步骤展开说明

#### Step ⑤：数学排布（所有项目通用）

**输入：** Step ④ 通过的配平数据（各功能块面积、尺寸、间数）

**前置条件（进入 Rhino 前必须完成）：**

1. 功能块清单定稿：公建/客房/配套 每块的尺寸(w×d)、间数、层数、每层高、总高
2. 数学排布定稿：每块的 XY 坐标、Z 高度通过 Python 计算确认，无重叠、间距合规
3. 面积预验算：用 Python 根据排布后的尺寸计算实际 GFA、基底面积，对 Step ④ 的目标值
4. 高度预验算：所有块最高点 ≤ 限高
5. 图层色码清单定稿：每个功能块绑定的图层名、RGB 色码
6. 生成 `_rhino_gen.py`：将以上所有数据写入可执行的 Rhino Python 脚本

**以上 1-5 条全部在 Python 中完成，确认无误后再执行第 6 步写入脚本。Rhino 只负责执行和最终验证，不负责排布计算。**

**操作：**

1. **确定坐标系统**
   - 所有功能块在同一坐标系下排列
   - 从左到右（X轴正向）按功能分类排列
   - Y轴用于同类功能块的多行排列

2. **功能块定位**
   - 同类功能块使用行列公式：
     ```
     cx = x0 + col * (块宽 + 列间距) + 块宽/2
     cy = y0 + row * (块深 + 行间距) + 块深/2
     ```
   - 不同类功能块之间留 ≥5000mm（5m）间距
   - 同一类功能块之间：联排紧邻间距4000mm，独栋间距3000mm

3. **高度赋值**
   - B1地下层: z = -1500, h = 3000（不计入建筑高度）
   - 1层(温泉/大堂等大空间): z = 2500, h = 5000
   - 2层以上(会议/餐饮): z = (下层顶) + 250, h = 4500
   - 独立体块: z = h/2（中心对齐）
   - 不超过限高（山地度假酒店≤12m）

**输出：** 所有功能块的坐标清单（cx, cy, cz, w, d, h）

---

#### Step ⑥：分类图层 + 色码（所有项目通用）

**输入：** Step ⑤ 的功能块坐标清单

**操作：**

1. **图层结构**
   ```
   R{轮次}
   ├── {方案字母}
   │   ├── {功能1}
   │   ├── {功能2}
   │   └── {功能3}
   ├── {方案字母2}
   └── {方案字母3}
   ```
   - `R1/A/公建B1_后勤`, `R1/B/公建1F_温泉SPA`, `R1/C/独栋客房`
   - 代码实现：`rs.AddLayer("A", parent="R1")` 然后 `rs.AddLayer("公建B1_后勤", parent="R1::A")`
   - `R` 从 1 开始递增（新项目或新轮次 +1）
   - 方案字母：A/B/C
   - 功能名：纯中文+下划线

2. **色码对照表**

   | 功能 | RGB | 色名 |
   |:----|:----|:-----|
   | 公建B1_后勤 | (0,100,180) | 深蓝 |
   | 公建1F_温泉SPA | (180,100,200) | 紫 |
   | 公建2F_大堂会议 | (0,150,230) | 蓝 |
   | 公建3F_餐饮 | (232,168,56) | 橙 |
   | 侧翼/附属 | (74,217,163) | 绿 |
   | 独栋客房 | (160,120,80) | 棕 |
   | 双拼客房 | (200,170,130) | 浅棕 |
   | 联排客房 | (220,195,160) | 米色 |

3. **编码实现**
   ```python
   rs.AddLayer("A", parent="R1")
   rs.AddLayer("公建1F_温泉SPA", parent="R1::A")
   # ...
   rs.LayerColor("R1::A::公建1F_温泉SPA", (180,100,200))
   ```

**输出：** Rhino 中完整图层树 + 每块按色码着色

---

#### Step ⑦：Rhino 生成 + 校验（所有项目通用）

**输入：** Step ⑤ 的坐标 + Step ⑥ 的图层

**操作：**

1. **生成 _rhino_gen.py**
   - 放在项目根目录（方便运行）
   - 格式：
     ```python
     import rhinoscriptsyntax as rs
     rs.DeleteObjects(rs.AllObjects())      # 清空旧模型（仅第一轮）
     for ln in rs.LayerNames():
         try: rs.DeleteLayer(ln)
         except: pass

     def bx(cx,cy,cz,w,d,h):
         hw=w/2;hd=d/2;hh=h/2
         return rs.AddBox([8个角点坐标])

     def mk(ly,nm,x,y,z,w,d,h,r,g,b):
         rs.CurrentLayer(ly)
         o=bx(x,y,z,w,d,h)
         if o: rs.ObjectColor(o,(r,g,b)); rs.ObjectName(o,nm)
     ```

2. **关键代码细节**
   - **坐标传递**：每个体外块由这组参数定义：`(图层,名称, cx,cy,cz, w,d,h, r,g,b)`
     - `cx,cy,cz` = 体块中心坐标（毫米）
     - `w,d,h` = 宽/深/高（毫米）
     - `r,g,b` = RGB 色码
   - **所有数据来自 Step ⑤ 计算结果**，不现场计算
   - **图层创建**：
     ```python
     rs.AddLayer("A", parent="R1")
     rs.AddLayer("公建1F_温泉SPA", parent="R1::A")
     rs.LayerColor("R1::A::公建1F_温泉SPA", (180,100,200))
     ```
   - **红线/退线**：用 `rs.AddPolyline` 从红线图坐标绘制场地边界
   - **命名规则**：`A_独栋01`（方案字母_功能+编号），前缀用于区分方案

3. **执行方式**
   - 通过 Rhino MCP 执行：`execute_rhinoscript_python_code`
   - 或直接在 Rhino 中用 `RunPythonScript` 运行

4. **校验 ①：重叠检测**
   - 遍历所有体块的 bounding box，两两对比 XY 投影是否重叠
   - 公建叠放（同一XY不同Z）跳过内部检测
   - **通过标准：** 跨方案 XY 重叠对数 = 0

5. **校验 ②：高度校验**
   - 遍历所有体块的 bounding box，取所有 Z_max 的最大值
   - **通过标准：** 最高点 ≤ 项目限高

6. **校验 ③：重复名称检查**
   - 所有体块命名必须唯一
   - **通过标准：** 无重复名称

7. **校验 ④：退线检查**
   - 验证每个体块的 4 个角点在退线范围内
   - **通过标准：** 0 个体块超出退线

8. **校验 ⑤：面积回检（硬性要求，不通过则退回 Step ⑤）**
   - 从 Rhino 模型读取每个体块的 bounding box，计算实际面积
   - 将每块实测面积 vs Step ④ 的目标面积
   - **要求：** 每类功能块总面积误差 ≤ ±5%，总 GFA 误差 ≤ ±5%
   - **不符时：** 禁止继续，必须退回 Step ⑤ 重新排布

9. **目录整理**
   - `_rhino_gen.py` → 项目根目录（已存在）
   - 源代码文件 → `src/` 目录
   - 输出文件（报告/JSON） → `output/` 目录
   - 旧模型/临时文件 → `_archive/` 目录

**输出：** Rhino 中 85 个体块、0 重叠、高度合规
