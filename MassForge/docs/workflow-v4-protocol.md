# MassForge 概念设计工作流 — 完整流程协议 V4.0

> 本工作流的设计目标是：**任何任务书 + 红线图 + 设计规范** 输入后，AI 自动完成从分析到建模的全流程。
> 不预设建筑形态，Step 2 的诊断结果决定一切。

---

## 工作流总图

```
① 环境准备（防重叠）
    │
    ▼
② 读输入（任务书 + 红线图 + 设计规范）
    │
    ▼
③ 类型学诊断 ← 核心决策层
    ├─ 判断建筑类型
    ├─ 确定设计矛盾与约束
    └─ 选择母题（决定生成策略）
    │
    ▼
④ 面积配平 + 规范校验（先算后建，不过停）
    │
    ▼
⑤ 创建图层（R轮次_方案字母_方案名）
    │
    ▼
⑥ 生成体块 ← 按类型诊断结果选择生成模板
    │
    ▼
⑦ 生成后校验 + 后合理化说明
```

---

## 各步骤详细说明

### Step ①：环境准备

目标：确定本次生成的位置和轮次，不与已有模型重叠。

- 读 Rhino 文档摘要，获取已有模型的边界框
- 读 `project_config.json`，获取场地红线尺寸
- 如果红线范围内已有体块：
  - 计算偏移量 = 红线最右 X 坐标 + 20m
  - 轮次编号 R = 已有最大轮次 + 1
- 如果无体块：
  - 偏移 = 0（从原点开始）
  - 轮次编号 R = 1
- 新建干净 .3dm 文件（可选）

### Step ②：读取输入

读取三个来源的结构化数据：

**任务书（docx）：**
- 总建筑面积（地上/地下）
- 建筑层数（地上/地下）
- 各层功能分配
- 客房/房间数（如适用）
- 限高
- 用地面积

**红线图（PDF/图片）：**
- 场地长宽尺寸
- 退线距离（东/西/南/北）
- 场地形状（规则/不规则）
- 周边道路关系

**设计规范（如 JGJ62-2014 等）：**
- 适用的规范条款
- 关键约束（走廊宽、客房面积等）

→ 结构化存储为 `project_config.json`

### Step ③：类型学诊断

**这是整个工作流的决策核心。** 诊断输出直接影响 Step ⑤ 的图层面结构和 Step ⑥ 的生成模板选择。

必须在读完输入后输出以下诊断报告，才能进入下一步：

```
=== 类型学诊断报告 ===
建筑类型:          [酒店/公寓/度假村/办公/综合体/文化/混合]
定位级别:          [经济型/中档/中高档/奢华]
核心设计矛盾:      [至少列出 2 条，从任务书分析得出]
功能组团划分:      [3-4 个组团，不预设垂直或水平]
推荐母题:          [2-3 个，按优先级排序]
推荐生成模板:      [塔楼类/水平展开类/分散类/大空间类/其他]
```

**诊断依据（来自 `.rhino-rules.md`）：**

| 类型 | 行业公理 | 核心矛盾锚点 |
|:----|:--------|:-----------|
| 商务酒店 | 下公上私，下闹上静 | 垂直声学阻尼；四流线分流；核心筒垂直交通效率 |
| 高端公寓 | 面宽最大化，户户朝南 | 日照阴影线控制；开间进深比 |
| 野奢度假村 | 景观最大化，分散式布局 | 地形等高线单向展开；视线扇面 |
| 总部办公 | 高效标准层，灵活分隔 | 核心筒偏置或外挂；标准层进深 |
| 商业综合体 | 多业态水平串联 | 人流动线组织；中庭空间序列；后勤与客流绝对隔离 |
| 文化建筑 | 空间叙事性 | 大跨度结构；流线引导；自然采光 |

**母题选择逻辑（`.rhino-rules.md` §2.2）：**

```
IF 酒店 AND 限高严格     → 推荐 Monolith, Terrace, Interlock
IF 公寓 AND 面宽要求高    → 推荐 Terrace, Monolith（排除 Interlock）
IF 度假村 AND 低密度分散  → 推荐 Terrace（排除 Monolith, Interlock）
IF 办公 AND 标准化要求高   → 推荐 Monolith, Interlock
IF 综合体 AND 地标性要求高 → 推荐 Interlock, Terrace
IF 文化建筑               → 推荐 Interlock, Monolith
```

**生成模板选择逻辑：**

| 诊断类型 | 生成模板 | 特征 |
|:--------|:---------|:------|
| 塔楼类（酒店/办公/公寓） | 垂直塔楼模板 | 裙房+塔楼+核心筒贯通+标准层重复 |
| 水平类（综合体/会展） | 水平展开模板 | 多翼裙房+中庭+连廊，无核心筒 |
| 分散类（度假村/别墅） | 分散布局模板 | 独立栋沿景观面展开，各自独立 |
| 大空间类（文化/体育/展馆） | 大空间模板 | 单层或多层大跨空间，异形轮廓 |
| 混合类（含多种业态） | 混合模板 | 组合上述策略 |

### Step ④：面积配平 + 规范校验

**铁律：必须先通过校验，才能进入建模。FAIL 必须停止。**

- 跑 `balance.py`（通用面积配平公式）
  - 输入：任务书参数
  - 输出：各标准层面积 + 误差率
  - 要求：误差 ≤ ±5%
- 跑 `validate.py`（规范校验引擎）
  - 逐层校验：每层面积 vs 目标
  - 规范校验：密度/容积率/限高/其他规范条款
  - 要求：全部 PASS
- 跑三方案对比
  - A/B/C 三个方案的面积/指标对比表
- **任何一项 FAIL → 报告错误，不建模**

### Step ⑤：创建图层

图层结构 = R{轮次}_{方案字母}_{方案名}，不预设"塔楼"相关命名。

```
R1
├── A_方案A名称         （唯一母题）
│   ├── 子层_功能区1
│   ├── 子层_功能区2
│   └── 子层_功能区3
├── B_方案B名称         （唯一母题）
└── C_方案C名称         （唯一母题）
```

规则：
- 图层名不带 `::` 特殊字符，用纯中文 + 下划线
- 子图层用 `ParentLayer()` 设置在父层下
- 图层名反映方案差异，不写死固定格式
- 每个方案一种母题，禁止混用

### Step ⑥：生成体块

根据 Step ③ 诊断出的**生成模板类型**，加载对应策略：

**垂直塔楼模板（如本次商务酒店）：**
- 红线 + 退线（从红线图读取坐标）
- 裙房体块（按功能分层）
- 塔楼体块（办公/客房等标准层）
- 核心筒贯通
- 按母题规则做几何调整

**水平展开模板（如综合体）：**
- 红线 + 退线
- 各裙房翼展开（按功能分区）
- 中庭空间
- 连廊连接

**分散布局模板（如度假村）：**
- 红线 + 退线
- 独立栋沿景观方向展开
- 每栋独立体块

**其他模板同理。**

所有体块必须：
- 在退线范围内
- 按色码着色
- 有命名
- 归入对应子图层

### Step ⑦：生成后校验 + 报告

- 读取 Rhino 模型体块的 BoundingBox
- 对比每个体块面积 vs 目标面积
- 偏差 > ±5% → 标记
- 输出后合理化说明（Post-Rationalization）：
  - 每个体块动作 → 对应的任务书条款
- 更新 dev-logs

---

## 铁律

1. **类型学优先**：Step ③ 未完成（无诊断报告），禁止进入 Step ④
2. **先算后建**：Step ④ FAIL，禁止进入 Step ⑤
3. **防重叠**：Step ① 必须检查模型位置，偏移后再生成
4. **轮次递增**：每次新生成 R 编号 +1，旧模型保留
5. **母题纯净**：一个方案只能使用一种母题，禁止混用
6. **二次失败停机**：连续两次生成失败 → 停止，输出错误分析
7. **后合理化**：Step ⑦ 不是可选项，每个体块动作须对应任务书条款
