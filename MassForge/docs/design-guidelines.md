# MassForge 概念设计工作流 — 设计规范

## 设计原则

1. **类型学优先**：任何几何生成之前必须经过类型学诊断，禁止跳过推理直接建模
2. **组团化思维**：建筑体块按功能归并为 2-4 个巨型组团，不逐层堆叠
3. **可验证性**：每个体块动作必须有对应的任务书条款或规范依据
4. **差异最大化**：同一任务书的三个方案必须在布局策略上有明显差异

## 类型学诊断框架

### 诊断输出规范

```
建筑类型:            [酒店/公寓/度假村/办公/综合体/...]
定位级别:            [经济型/中档/中高档/奢华]
核心设计矛盾:        [至少列出 2 条]
功能组团划分:        [3-4 个巨型组团]
推荐适用母题:        [2-3 个，按优先级排序]
```

### 建筑类型与核心矛盾

| 类型 | 行业公理 | 核心矛盾 |
|:----|:--------|:---------|
| 商务酒店 | 下公上私，下闹上静 | 垂直声学阻尼、四流线分流、核心筒效率 |
| 高端公寓 | 面宽最大化，户户朝南 | 日照阴影线控制、开间进深比 |
| 野奢度假村 | 景观最大化，分散式布局 | 地形等高线单向展开、视线扇面 |
| 总部办公 | 高效标准层，灵活分隔 | 核心筒偏置、标准层进深 |
| 商业综合体 | 多业态水平串联 | 人流动线、中庭序列、客流后勤隔离 |

## 母题语法体系

### Monolith（极简巨构）
- 外立面 100% 垂直平齐，整体性为最高准则
- 面积差通过内部中庭/入口凹洞/天井消化
- 适用：城市核心区、限高严格、高层建筑

### Terrace（阶梯错叠）
- 多个组团间整体性跌落，退台单向展开
- 退让幅度 ≥3m/次，退台面自动标记为 Terrace_Zone
- 适用：沿主干道、有景观面要求、需公共空间补偿

### Interlock（异质穿插）
- 裙房与塔楼独立几何中心轴，Z 向允许物理交叉
- 悬挑 ≤3.0m 常规 / ≤6.0m 云顶标志性悬挑
- 适用：地标性项目、转角地块、不限高

## V3.0 特化：方案 B/C 生成规则

### 方案 B — Terrace（城市绿毯）
- **东南角锁定**：东南角点刚性锁定，东/南界面 1F-26F 绝对垂直平齐
- **塔楼高宽比**：`tower_aspect_ratio` ≥ **1.6:1**
- **单向西侧大退台**：4F 顶面向西（内院）组团叠落，退台深度 ≥**15m**

### 方案 C — Interlock（极简双折体）
- **晶体合并**：5F-25F（含16F）完全合并为一个完整晶体，取消该区段所有退台
- **非对称咬合**：晶体塔楼与基座错位，底层可架空
- **云顶硬核反冲**：26F 沿主干道单向悬挑 ≥**6.0m**

## 图层规范

每个方案一个父图层 A/B/C，子图层按功能划分：

```
A（方案A）
├── 控制线
├── 裙房
│   ├── 1F_功能名（色码）
│   ├── 2F_功能名（色码）
│   └── ...
├── 办公（功能组团 2）
├── 酒店客房（功能组团 3）
├── 核心筒
└── 标注
```

## 铁律检查清单

- 单位使用 mm
- 面积配平误差 ≤ ±5%
- 所有体块在红线内
- 塔楼在裙房范围内（四周内缩 ≥3m）
- 核心筒在塔楼内部（不在边缘）
- 同一组团内部 100% 垂直平齐
- ABC 布局明显不同
- 颜色 RGB 正确

## 后合理化说明规范

每个体块动作完成后，必须说明对应的任务书条款：

```
[体块动作] → [任务书条款]
  具体说明

例：
16F 四周内缩 → 客房标准层从 1,350㎡ 退台至 1,210㎡
  → 退台面形成 16F 麻将客房的专属空中花园
```
